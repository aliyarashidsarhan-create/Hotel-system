# Hotel Management System - User Stories & Product Backlog

| ID  | User Story | Priority | Complexity | Story Points | Acceptance Criteria |
|-----|------------|----------|------------|--------------|-------------------|
| US01 | As a guest, I want to search available rooms so that I can choose a room to book. | High | Medium | 5 | Guest can view available rooms for selected dates; room types and prices are displayed. |
| US02 | As a guest, I want to book a room so that I can stay at the hotel. | High | High | 8 | Booking is confirmed with a unique booking ID; guest receives confirmation email. |
| US03 | As a guest, I want to cancel my booking so that I can change my plans if needed. | Medium | Medium | 5 | Booking can be canceled before check-in; refund policy is applied automatically. |
| US04 | As a guest, I want to view my booking history so that I can keep track of past stays. | Medium | Low | 3 | Guest can see all past bookings with dates, room type, and price. |
| US05 | As a hotel admin, I want to add new rooms so that the hotel inventory is updated. | High | Medium | 5 | Admin can add room details including type, price, and availability; changes are saved. |
| US06 | As a hotel admin, I want to update room details so that pricing and availability are accurate. | Medium | Medium | 5 | Admin can update room info; updates reflect immediately for guests. |
| US07 | As a hotel admin, I want to remove rooms so that unavailable or retired rooms are not bookable. | Medium | Medium | 5 | Removed rooms are no longer visible to guests; system confirms deletion. |
| US08 | As a guest, I want to make online payments so that I can confirm my booking immediately. | High | High | 8 | Payment gateway works securely; booking confirmed only after successful payment. |
